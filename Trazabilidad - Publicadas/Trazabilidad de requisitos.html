Sobre los ID's
El formato siempre debe ser el siguiente:
Rxxx.SRxxxx.Fxxxx.SFxxx.Pxxx.SPxxx.ECxxx
La x corresponde a un número entre 0 y 9
Cuando no hay una parte asociada a un elemento, se coloca 000 para las partes no pertinentes. ej:
Requisito
R001.SR00.F000.SF000.P000.SP000.EC000 Como solo existe el requisito, los demas espacios de id quedan en 0
Sub-requisito
R001.SR001.F000.SF000.P000.SP000.EC000 Como solo existe el requisito, los demas espacios de id quedan en 0
Funcion R001.SR001.F001.SF000.P000.SP000.EC000 El espacio de IDs validos solo cubre a la funcion y sus padres.
Sobre el manejo de dependencias
En el registro de dependencias puede haber muchas instancias de tareas, y puede haber varios tipos de ellas con diversos formatos. Siempre debe respetarse el siguiente orden en el registro de dependencias:
ID Origen Fecha de asignación Descripción
Rxxx.SRxxx.Fxxx.SFxxx.Pxxx.SPxxx.ECxxx Otro elemento de este archivo N/A
xxxxxx Una tarea de una minuta relacionada con una reunión de seguimiento. Utilizar el ID de la tarea Fecha de la minuta: dd/mm/aaaa
